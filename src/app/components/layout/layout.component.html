Responsive layout grid that automatically adapts to screen size and orientation ensuring an optimal experience across smartphones, tablets, and desktop computers. The layout module consists of the Layout Service and a set of visual layout components:

<ea-drawer>

(optional) panel that expands/collapses from one side of the screen. The ea-drawer is designed to have a ,<header>, <section>, and <footer>. A <header> element will always sit at the top of the drawer. The <section> is stretched to be the space between the header and the footer aligned at the top. The <footer> is aligned at the bottom always. For simplicity sake we accomplish this we flexbox which makes it ideal to only have 3 elements, header, section, and footer to maintain this layout easily.

The drawer uses different an event stream to inform the application what state it is in. The first two states we've introduced are maximized and closed. We plan to introduce new states to make it so that you can adjust the size of your drawer to different states we think have utility.
