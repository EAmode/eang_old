Reactive

Status changes in enterprise applications need to be updated in real-time. The API design of eang is conducive to reactive functional programming and event streams. Eang components support both push and pull based models.

The following example shows the ea-panel component reacting to two event stream observables, panel state and orientation.

<ea-panel>

({{ panelOrientation | async }}, {{ panelState | async }})

ea-panel

These UI components interact with each other through composable event streams. This allows developers to implement more rapidly and reach a higher level of interaction complexity in their applications.
