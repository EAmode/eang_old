CSS Themes

Styling and theming in eang is fully customizable and can be easily changed. New and upcoming features of CSS alleviate a lot of the maintainability problems of CSS. Eang relies heavily on CSS Custom Properties and nesting and uses a PostCSS pipeline to support older browsers.

Unlike other component frameworks, the style is not set in stone and does not abstract or encapsulates away the power of CSS. Instead it embraces new capabilities of CSS and puts design back into the hand of the designer.

Eang has multiple build-in themes. Each theme can be extensively customized with CSS Custom Properties. Each theme has global variables as well as component based CSS variables.

These properties can also just apply within a specific scope of your page.

If overriding variables and properties for some reason does not work in your case, there is nothing prevents you from creating your own theme.

If that is not enough, eang allows you to define the CSS of each component individually additional themes can be be added. This allows users of eang to fully customize the the look and feel of each component and makes it easy to comply with Corporate Design guidelines.
